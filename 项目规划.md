# 新闻链接展示 Web 应用 - 项目规划

## 一、项目背景

每周向同事推送 10-15 条新闻链接，涉及五个固定板块（客户动向、友商观察、行业脉搏、全球视野、科技前沿）。目前缺乏集中展示与筛选手段，需构建一个轻量级 Web 应用，实现新闻的集中展示、多维筛选和便捷更新。

**核心诉求：** 通过上传 Excel 文件完成每周内容更新，同事访问时按公司/主题/年份/月份/区域五维筛选浏览，同时设置访问密码保护。

**用户角色：**
- **管理员**（1人）：负责每周通过 Excel 导入更新新闻数据，拥有管理入口权限
- **浏览者**（多人）：同事，通过密码验证后浏览和筛选新闻，无数据修改权限

---

## 二、功能模块

| 模块 | 功能 | 角色 | 优先级 |
|------|------|------|--------|
| **新闻展示** | 主页以栏目卡片网格展示（每个栏目2-3条），点击more进入栏目详情页（每页10条、分页） | 浏览者 | P0 |
| **多维筛选** | 支持按公司、主题、年份、月份、区域五维筛选 | 浏览者 | P0 |
| **Excel 导入** | 上传 Excel 文件解析并更新新闻数据 | 管理员 | P0 |
| **访问控制** | 密码验证后访问，会话保持 24 小时 | 全部 | P0 |
| **响应式布局** | 移动端/桌面端自适应 | 全部 | P1 |

## 三、数据设计

### 3.1 Excel 导入模板

| 字段 | 说明 |
|------|------|
| 栏目 | 五板块之一：客户动向/友商观察/行业脉搏/全球视野/科技前沿 |
| 标题 | 新闻标题 |
| 链接 | 新闻 URL |
| 公司 | 涉及的公司/机构名称 |
| 主题 | 关键词标签，多个用英文逗号分隔 |
| 区域 | 地区分类（华东/华北/华南/华中/西南/西北/东北/海外/国内等，从数据中提取） |
| 发布 | 第一行：日期（YYYY年MM月DD日）；第二行：期数（第 xx 期） |

          "company": "公司名",
          "tags": ["标签1", "标签2"],

    }

### 4.2 空状态与异常设计

| 场景 | 处理方式 |
|------|----------|
| 未导入数据 | 显示空状态引导：📡 暂无新闻数据，提示点击 ⚙️ → 📥 导入 Excel 文件 |
| 筛选无结果 | 显示"暂无相关新闻"提示 |
| 导入数据校验失败 | 逐条提示错误字段，允许修正后重新导入 |
| 新闻链接失效 | 点击后由浏览器报错，页面不做额外拦截 |
| 密码输错 | 提示"密码错误"，连续 5 次错误后锁定 15 分钟 |
| 数据加载失败 | 显示"数据加载异常，请刷新页面"提示 |
| 筛选异常 | 页内诊断面板（⚙️ → 🔍 诊断）可排查缓存状态、数据、筛选匹配问题 |

### 4.3 技术选型

| 层面 | 方案 | 说明 |
|------|------|------|
| 框架 | 纯静态（HTML5 + CSS3 + JS ES6+） | 无需后端，部署简单 |
| Excel 解析 | SheetJS（xlsx） | 前端解析 Excel 文件 |
| 密码保护 | SHA-256 哈希 + Web Crypto API | 前端验证，LocalStorage 会话保持 |
| 数据存储 | JSON 文件 + LocalStorage 缓存 | 导入后生成 news.json |
| 样式 | CSS Grid/Flexbox | 响应式布局 |

### 4.4 安全边界说明

> 本应用采用纯前端密码保护，定位为**使用门槛**而非**安全防护**。技术用户可通过浏览器开发者工具直接访问 news.json 文件绕过密码验证。如需真正的数据安全，应采用后端服务方案。当前方案适用于内部团队场景，目的在于避免无关人员随意访问，而非防止恶意攻击。

## 五、关键流程

### 5.1 访问流程
```
用户访问 → 检查会话 → 无会话：显示密码页 → 输入密码 → SHA-256验证 → 写入会话 → 进入主页
                        → 有会话：直接进入主页
```

### 5.2 Excel 导入流程
```
管理员点击导入 → 选择Excel文件 → SheetJS自动解析并转换为JSON → 字段映射（发布列拆分为日期+期数）
→ 数据校验（必填字段、URL格式） → 去重（基于URL） → 自动保存到 LocalStorage → 前端页面实时更新展示
```
> ✅ **自动转换机制：** Excel → JSON 的转换由前端 SheetJS 库在后台自动完成，转换后的数据自动保存到浏览器 LocalStorage 并实时更新前端展示，管理员无需手动下载或替换任何文件。

### 5.3 筛选流程
```
选择筛选条件 → 遍历news.json → 按公司/主题/年份/月份/区域过滤 → 重新渲染列表
（五个筛选条件取交集；年份/月份从 publishDate 中提取；公司/主题/区域均从数据中的对应字段提取）
```

## 七、部署方案

| 方案 | 适用场景 | 说明 |
|------|---------|------|
| **GitHub Pages** | 远程访问、免费部署 | 推荐，自动 HTTPS，全球可访问 |
| **阿里云轻量服务器** | 国内访问、低成本运维 | 约 38-68 元/年（2核2G，200M 峰值带宽，40-50G 存储），适合国内团队使用 |
| 本地局域网 | 团队同办公地点 | 简单快速，无需域名 |
| 企业内网服务器 | 数据安全要求高 | 可控性强 |

### 数据更新流程

```
管理员打开Web应用 → 点击导入按钮 → 选择本周Excel文件
→ 前端自动解析Excel并转换为JSON格式 → 自动保存到 LocalStorage → 前端实时更新展示
```

**维护要点：** 每周 Excel 导入更新；每月更换访问密码；每月检查链接有效性。

### 分阶段部署策略

| 阶段 | 方案 | 说明 |
|------|------|------|
| **第一阶段：体验验证** | GitHub Pages（免费） | 开发完成后部署到 GitHub Pages，免费分享给同事体验验证 |
| **第二阶段：正式部署** | 阿里云轻量服务器（38-68 元/年） | 体验通过后申请公司预算，升级至阿里云轻量服务器，部署到国内节点，提升访问速度 |

### 后续升级规划

> 当前版本使用 LocalStorage 存储，数据仅保存在当前浏览器中，适合测试和单设备使用。完成开发后，先通过 GitHub Pages 免费部署供同事体验，体验通过后申请公司预算，升级为阿里云轻量服务器方案（约 38-68 元/年），部署静态网站并托管 JSON 数据，实现多设备/多浏览器访问，国内访问速度快，运维简单。
